The central new system of the Gay Blade's **shiv-style build**. Captures Hallie's pivot
(2026-06-15, recorded in bro-engine): the modifier engine is for **shivs / throwing shivs**,
**NOT arrows** — there is no Arrow card or archetype. The mechanics that once read as bow-words

## What a "shiv" is here
- The base game **`Shiv`** (`.decompiled/.../Models/Cards/Shiv.cs`): 0-cost Token Attack,
  deal 4, Exhaust, goes AoE under `FanOfKnivesPower`. Upgrades +2. Generated by Silent Pride,
  Superfan of Knives, Knife Whip.
- The mod's **Throwing Shiv** (`Kunai.cs`, unified 2026-06-15): 0-cost; played deals BASE-2 = 1;
  held to end of turn deals BASE = 3 AoE and Exhausts. BASE_DAMAGE = 3.

## Engine path (the *can* — ready for ratification)
BaseLib already ships the substrate: **`CardModifier`** (`.decompiled-baselib/Baselib/Abstracts/CardModifier.cs`).
A `CardModifier` is a per-card attachment with exactly the hooks this needs:
- `OnPlay(choiceContext, cardPlay)` — run extra effects when the modified shiv is played
  (Poison application, Pin's Weak+Vulnerable, Explosive Tip's AoE).
- `ModifyDescription` / `ModifyDescriptionPost` — make the shiv's text read true when modified.
- save/load via `ModifierSave` (IntProperties / AdditionalProperties) — persists across save/quit.
- `ShouldReceiveCombatHooks => true` + sorted `Priority` — so damage hooks (Long Bow pierce,
  Ricochet spill) can ride the AbstractModel damage pipeline like the blades already do.
- Apply with `CardModifier.AddModifier(card, modifier)`; query with `CardModifier.Modifiers(card)`.

Two viable shapes (Hallie picks):
1. **`// PROPOSAL` Per-shiv modifier.** A turn-wide "Poison Coating" effect, when triggered,
   walks every shiv in hand/pile and `AddModifier`s a `PoisonCoatingMod` to each. Cleanest for
   "this turn your shivs deal Poison." Pierce/spill live as damage hooks on the modifier.
2. **`// PROPOSAL` Player-power buff + shiv self-check.** A `ShivBuffPower` on the player; each
   shiv checks `Owner.HasPower<...>()` in its own `OnPlay`/damage hooks (the way the base Shiv
   already checks `FanOfKnivesPower` for AoE). Less per-card bookkeeping; matches existing pattern.

The base Shiv's own `HasFanOfKnives`→AoE check is the **proof** that shape 2 works in-engine and
is the lowest-risk path; `Explosive Tip` (AoE) is *literally* the FanOfKnives pattern already in
`SuperfanOfKnives.cs` / `FanOfKnivesThisTurnPower`.

## Relationship to the rest of the deck (don't double-build)
- `FanOfKnivesThisTurnPower` already does "shivs hit all enemies this turn" — **Explosive Tip's
  AoE half is already built**; only the Exhaust-this-turn rider is new.
- Silent Pride / Superfan / Knife Whip already generate shivs — those are the **Quiver** feeders;
  Quiver may be a *naming/relic* over the existing generators rather than new code. Hallie to say.
